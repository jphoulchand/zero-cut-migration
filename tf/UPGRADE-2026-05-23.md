## Deployment Notes

### Pre-Upgrade Checklist

- [x] Backup terraform state
- [x] Review breaking changes (none identified)
- [x] Update addon versions
- [x] Update Helm chart versions
- [x] Update Terraform provider versions

### Deployment Order

1. **Terraform Init** (downloads new providers)
   ```bash
   terraform init -upgrade
   ```

2. **Terraform Plan** (review changes)
   ```bash
   terraform plan
   ```

3. **Terraform Apply** (staged deployment)
   ```bash
   # Apply in stages for safety
   terraform apply -target=module.eks
   terraform apply -target=aws_eks_addon.vpc_cni
   terraform apply -target=aws_eks_addon.coredns
   terraform apply -target=helm_release.karpenter
   terraform apply  # Apply remaining
   ```

### Expected Downtime

- **EKS Control Plane:** ~15 minutes (cluster remains available)
- **Addons:** Rolling updates, no downtime
- **Karpenter:** Brief interruption during upgrade
- **Nodes:** Rolling replacement (no downtime with proper pod disruption budgets)

### Rollback Plan

If issues occur:

1. **Revert terraform.tfvars:**
   ```hcl
   kubernetes_version = "1.33"
   ```

2. **Revert versions.tf:**
   ```hcl
   aws = { version = ">= 6.23.0" }
   kubernetes = { version = "~> 3.0" }
   ```

3. **Revert main.tf addon versions** (keep backup file)

4. **Apply:**
   ```bash
   terraform apply
   ```

**Note:** EKS clusters cannot be downgraded, but you can prevent further upgrades.

## Verification Steps

### Post-Upgrade Checks

```bash
# 1. Verify cluster version
aws eks describe-cluster \
  --name <cluster-name> \
  --query 'cluster.version' \
  --output text
# Expected: 1.35

# 2. Check addon versions
aws eks list-addons \
  --cluster-name <cluster-name> \
  --output table

aws eks describe-addon \
  --cluster-name <cluster-name> \
  --addon-name vpc-cni \
  --query 'addon.addonVersion' \
  --output text
# Expected: v1.21.2-eksbuild.2

# 3. Verify nodes
kubectl get nodes
# All nodes should be Ready

# 4. Check Karpenter
helm list -n karpenter
# Version: 1.12.1

kubectl get nodepools
# Should list all 3 nodepools

# 5. Verify CoreDNS
kubectl get pods -n kube-system -l k8s-app=kube-dns
# All pods Running

# 6. Test DNS resolution from jumpbox
ssh -i <key> ec2-user@<jumpbox-ip>
dig kafka.confluent.svc.cluster.local +short
# Should resolve without errors

# 7. Verify Kafka connectivity
kafka-broker-api-versions \
  --bootstrap-server kafka.confluent.svc.cluster.local:9092
# Should show all brokers
```
