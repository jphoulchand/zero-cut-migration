# Confluent Kafka on Kubernetes with mTLS

Production-ready Kafka cluster deployment on Amazon EKS using Confluent for Kubernetes operator with mutual TLS authentication.

## Cluster Configuration

| Component | Version | Replicas | Resources |
|-----------|---------|----------|-----------|
| Confluent Operator | 3.2.2 (0.1514.40) | 2 | - |
| Confluent Platform | 8.2.1 | - | - |
| KRaft Controllers | 8.2.1 | 5 | 200m-1000m CPU, 2-4Gi RAM, 30Gi storage |
| Kafka Brokers | 8.2.1 | 12 | 500m-2000m CPU, 4-8Gi RAM, 50Gi storage |
| Schema Registry | 8.2.1 | 3 | 200m-1000m CPU, 1-2Gi RAM |
| Kafka Connect | 8.2.1 | 2 | 500m-2000m CPU, 2-4Gi RAM |

## Architecture

- **Mode**: KRaft (no ZooKeeper)
- **Security**: mTLS with auto-generated certificates
- **TLS Protocol**: TLS 1.3 only
- **Storage**: AWS EBS GP3 volumes
- **Node Architecture**: ARM64 (Graviton) spot instances
- **High Availability**: 
  - KRaft controllers: Preferred anti-affinity (5 replicas / 3 AZs = max 2 per zone)
  - Kafka brokers: Preferred anti-affinity (12 replicas / 3 AZs = max 4 per zone)
  - Replication factor: 3
  - Min in-sync replicas: 2

## Prerequisites

1. **EKS Cluster** with:
   - Kubernetes 1.35
   - Karpenter for node provisioning
   - ARM64 node support
   - Storage class: `gp3`

2. **Confluent for Kubernetes Operator** deployed:
   ```bash
   helm repo add confluentinc https://packages.confluent.io/helm
   helm repo update
   helm upgrade --install confluent-operator confluentinc/confluent-for-kubernetes \
     --version 0.1514.40 \
     --namespace confluent \
     --create-namespace
   ```

3. **CA Certificate** for TLS:
   - Generated in `certs/generated-certs/ca/`
   - Required for `autoGeneratedCerts` feature

## Deployment

### 1. Generate CA Certificate (if not exists)

```bash
cd certs
./generate-kafka-certs.sh
```

This creates:
- CA certificate: `generated-certs/ca/ca-cert.pem`
- CA private key: `generated-certs/ca/ca-key.pem`
- Broker certificates (not used with autoGeneratedCerts, but available if needed)

### 2. Create CA Secret

The Confluent operator uses this CA to auto-generate certificates for all components:

```bash
kubectl create secret tls ca-pair-sslcerts \
  --cert=certs/generated-certs/ca/ca-cert.pem \
  --key=certs/generated-certs/ca/ca-key.pem \
  -n confluent
```

### 3. Apply RBAC Configuration

Required for Kafka init containers to access Kubernetes API:

```bash
kubectl apply -f kafka-rbac.yaml
```

This creates:
- ServiceAccount: `kafka`
- Role: `kafka-role` (pods permissions)
- ClusterRole: `kafka-cluster-role` (nodes permissions)
- RoleBinding and ClusterRoleBinding

### 4. Deploy Kafka Core

Deploy KRaft controllers and Kafka brokers:

```bash
kubectl apply -f kafka-core.yaml
```

**Wait for all pods to be ready** (~3-5 minutes):

```bash
kubectl get pods -n confluent -w
```

Expected:
```
kraftcontroller-0   1/1   Running
kraftcontroller-1   1/1   Running
kraftcontroller-2   1/1   Running
kraftcontroller-3   1/1   Running
kraftcontroller-4   1/1   Running
kafka-0             1/1   Running
kafka-1             1/1   Running
...
kafka-11            1/1   Running
```

### 5. Verify Kafka Cluster

Check cluster status:

```bash
kubectl get kafka -n confluent
```

Expected output:
```
NAME    REPLICAS   READY   STATUS    AGE
kafka   12         12      RUNNING   5m
```

Check KRaft controllers:

```bash
kubectl get kraftcontroller -n confluent
```

Expected output:
```
NAME              REPLICAS   READY   STATUS    AGE
kraftcontroller   5          5       RUNNING   5m
```

### 6. Deploy Auxiliary Services (Optional)

Once Kafka core is verified, deploy Schema Registry and Connect:

```bash
kubectl apply -f kafka-auxiliary.yaml
```

## Security Configuration

### TLS Settings

- **Protocol**: TLS 1.3 only
- **Client Authentication**: Required (mTLS)
- **Endpoint Identification**: HTTPS
- **Certificate Management**: Auto-generated by Confluent operator

### Principal Mapping

Certificates are mapped to Kafka principals using:

```yaml
principalMappingRules:
  - "RULE:^CN=(.*?),OU=Engineering,O=Confluent,L=MountainView,ST=CA,C=US$/$1/"
  - "DEFAULT"
```

This extracts the Common Name (CN) from the certificate as the principal.

## Troubleshooting

### Pods Stuck in Pending

**Issue**: Kafka pods remain in `Pending` state

**Cause**: Karpenter not provisioning nodes or anti-affinity constraints

**Solution**:
```bash
# Check Karpenter logs
kubectl logs -n karpenter -l app.kubernetes.io/name=karpenter --tail=50

# Check pod events
kubectl describe pod kafka-0 -n confluent

# Verify NodePool limits
kubectl get nodepools -o yaml | grep -A 5 "limits:"
```

### Certificate Errors

**Issue**: `Secret "ca-pair-sslcerts" not found`

**Solution**:
```bash
# Verify CA secret exists
kubectl get secret ca-pair-sslcerts -n confluent

# Recreate if needed
kubectl create secret tls ca-pair-sslcerts \
  --cert=certs/generated-certs/ca/ca-cert.pem \
  --key=certs/generated-certs/ca/ca-key.pem \
  -n confluent
```

### Configuration Conflicts

**Issue**: `Only one of inter.broker.listener.name and security.inter.broker.protocol should be set`

**Cause**: Conflicting Kafka configurations in `configOverrides`

**Solution**: Remove `security.inter.broker.protocol=SSL` from configOverrides. The operator sets `inter.broker.listener.name` automatically when using mTLS.

### RBAC Errors

**Issue**: Init containers fail with permission denied errors

**Solution**: Ensure `kafka-rbac.yaml` is applied and pods use `serviceAccountName: kafka`

```bash
kubectl apply -f kafka-rbac.yaml
kubectl get serviceaccount kafka -n confluent
```

## Cost Optimization

- **Spot Instances**: Karpenter configured for spot-only ARM64 nodes
- **Instance Types**: Cheapest ARM64 instances prioritized (t4g.medium, t4g.large, etc.)
- **Auto-scaling**: Karpenter provisions nodes on-demand
- **Storage**: GP3 volumes (cheaper than GP2, better performance)

**Estimated monthly cost** (12 brokers + 5 controllers on spot ARM64):
- Compute: ~$150-200/month (spot pricing)
- Storage: ~$50-60/month (850Gi GP3)
- **Total**: ~$200-260/month

## Maintenance

### Scaling Brokers

Edit `kafka-core.yaml`:

```yaml
spec:
  replicas: 16  # Increase from 12
```

Apply:

```bash
kubectl apply -f kafka-core.yaml
```

### Upgrading Confluent Platform

1. Update image versions in `kafka-core.yaml` and `kafka-auxiliary.yaml`
2. Apply changes (rolling restart):
   ```bash
   kubectl apply -f kafka-core.yaml
   kubectl apply -f kafka-auxiliary.yaml
   ```

### Certificate Rotation

The operator auto-rotates certificates. Monitor certificate expiration:

```bash
kubectl get secret -n confluent -o jsonpath='{range .items[?(@.type=="kubernetes.io/tls")]}{.metadata.name}{"\n"}{end}'
```
