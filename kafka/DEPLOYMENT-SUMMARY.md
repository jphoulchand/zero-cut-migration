## Issues Encountered and Resolved

### 1. Manual Certificate Management

**Issue**: Initial approach used manually generated JKS keystores with jksPassword.txt files, causing mount path and password format conflicts.

**Resolution**: Switched to `autoGeneratedCerts: true` with operator-managed certificate lifecycle.

### 2. Configuration Conflict

**Issue**: Error: "Only one of inter.broker.listener.name and security.inter.broker.protocol should be set"

**Resolution**: Removed `security.inter.broker.protocol=SSL` from configOverrides. Operator sets `inter.broker.listener.name` automatically.

### 3. Pod Anti-Affinity Constraints

**Issue**: KRaft controllers stuck in Pending due to `requiredDuringSchedulingIgnoredDuringExecution` with 5 replicas across 3 AZs.

**Resolution**: Changed to `preferredDuringSchedulingIgnoredDuringExecution` to allow 2 controllers per zone.

### 4. Karpenter CPU Limits

**Issue**: Kafka pods pending due to insufficient CPU quota (1000 vCPUs per AZ was too low).

**Resolution**: Increased to 12,000 vCPUs per AZ in NodePool configuration.

### 5. Spot Instance Configuration

**Issue**: Not explicitly requesting cheapest instances.

**Resolution**: 
- Set capacity type to spot-only
- Ordered instance types by cost (t4g.medium first)
- Added broader range of ARM64 instance types
